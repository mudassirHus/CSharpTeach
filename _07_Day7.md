# Day 7: Arrays (Complete Beginner Guide)

Arrays are used to **store multiple values in a single variable** instead of creating many variables.

In simple words:
If you want to store **many values of the same type** → use an **array** 📦

---

## 1. What is an Array?

An **array** is a collection of elements of the **same data type**, stored in **continuous memory locations**.

> Each element is accessed using an **index** (position number).

---

## 2. Why do we need Arrays?

We use arrays to:

* Store multiple values efficiently
* Reduce number of variables
* Easily process large data
* Use loops effectively

Example without array ❌:

```chsharp
int m1 = 60;
int m2 = 70;
int m3 = 80;
```

Example with array ✅:

```csharp
int[] marks = {60, 70, 80};
```

---

## 3. Array Index

* Index starts from **0**
* Last index = size - 1

Example:

```csharp
int[] arr = {10, 20, 30, 40};
```

| Index | Value |
| ----- | ----- |
| 0     | 10    |
| 1     | 20    |
| 2     | 30    |
| 3     | 40    |

---

## 4. Declaring an Array

### Syntax:

```csharp
dataType[] arrayName;
```

### Example:

```csharp
int[] arr;
```

---

## 5. Creating an Array

### Syntax:

```java
arrayName = new dataType[size];
```

### Example:

```csharp
arr = new int[5];
```

---

## 6. Declaration + Creation + Initialization (Together)

```java
int[] numbers = {10, 20, 30, 40, 50};
```

---

## 7. Accessing Array Elements

```csharp
int[] arr = {10, 20, 30};
Console.WriteLine(arr[0]); // 10
Console.WriteLine(arr[2]); // 30
```

---

## 8. Modifying Array Elements

```csharp
int[] arr = {10, 20, 30};
arr[1] = 99;
Console.WriteLine(arr[1]); // 99
```

---

## 9. Array Length

Use `.Length` property to find array size.

```java
int[] arr = {10, 20, 30, 40};
Console.WriteLine(arr.Length); // 4
```

---

## 10. Traversing Array using for Loop

```csharp
int[] arr = {10, 20, 30};

for(int i = 0; i < arr.Length; i++)
{
    Console.WriteLine(arr[i]);
}
```

---

## 11. Common Beginner Mistakes

* Accessing invalid index (ArrayIndexOutOfBoundsException)
* Confusing index with value
* Using wrong loop condition
* Forgetting array size

Wrong ❌:

```csharp
System.out.println(arr[5]);
```

---

## 12. ArrayIndexOutOfBoundsException

Occurs when index is:

* Less than 0
* Greater than or equal to array size

---

## 13. Default Values in Array

| Data Type | Default Value |
| --------- | ------------- |
| int       | 0             |
| float     | 0.0           |
| char      | '\u0000'      |
| boolean   | false         |
| String    | null          |

---

## 14. When to use Arrays?

* Same type of data
* Fixed size data
* Faster access using index

---

## 15. 2D Arrays

### What is a 2D Array?

A **2D array** is an array of arrays. It is used to store data in **row and column** format (table form).

Example use cases:

* Matrix
* Marks of students (rows = students, columns = subjects)

### Syntax:

```csharp
int[,] matrix = new int[2,3];
```

### Example:

```csharp
int[,] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};
```

👉 Access element using two indexes: `matrix[row, column]`

---

## 16. Jagged Arrays

### What is a Jagged Array?

A **jagged array** is an array of arrays where **each inner array can have different sizes**.

Think of it as:

> A collection of 1D arrays with different lengths

### Syntax:

```csharp
int[][] jagged = new int[3][];
```

### Example:

```csharp
int[][] jagged = new int[3][];

jagged[0] = new int[] {1, 2};
jagged[1] = new int[] {3, 4, 5};
jagged[2] = new int[] {6};
```

👉 Each row can have **different number of elements**

---

## 17. Difference Between 1D, 2D, and Jagged Arrays

| Feature     | 1D Array    | 2D Array               | Jagged Array       |
| ----------- | ----------- | ---------------------- | ------------------ |
| Structure   | Single list | Table (rows & columns) | Array of arrays    |
| Size        | Fixed       | Fixed rows & columns   | Variable row sizes |
| Memory      | Continuous  | Continuous             | Non-continuous     |
| Access      | arr[i]      | arr[i, j]              | arr[i][j]          |
| Flexibility | Low         | Medium                 | High               |

---

## 19. Updated Key Takeaways

* 1D array → simple list
* 2D array → table structure
* Jagged array → flexible rows
* Arrays are foundation for advanced DSA

## 20. How Arrays are Stored in Memory (Stack vs Heap)

Understanding **memory allocation** is very important for interviews and deep clarity.

---

### Stack Memory

* Stores **local variables** and **references**
* Works in **LIFO** (Last In, First Out)
* Very fast
* Memory is automatically freed when method ends

Example:

```csharp
int x = 10;
```

Here:

* `x` is a value type
* Stored **directly in stack**

---

### Heap Memory

* Stores **objects and arrays**
* Memory allocated using `new`
* Slower than stack
* Managed by **Garbage Collector (GC)**

Example:

```csharp
int[] arr = new int[3];
```

Here:

* Array object is stored in **heap**
* Variable `arr` (reference) is stored in **stack**

---

### Visual Representation

```
Stack            Heap
-----            -----
arr  --------->  [10, 20, 30]
```

* Stack stores **address/reference**
* Heap stores **actual data**

---

### Important Points (Beginner Friendly)

* Arrays are **reference types** in C#
* Reference → stack
* Actual array → heap
* Multiple references can point to same heap array

---

### Example: Reference Behavior

```csharp
int[] a = {1, 2, 3};
int[] b = a;
b[0] = 99;

Console.WriteLine(a[0]); // 99
```

Reason:

* Both `a` and `b` point to **same heap memory**

---

### Stack vs Heap Summary Table

| Feature | Stack                 | Heap            |
| ------- | --------------------- | --------------- |
| Stores  | Variables, references | Objects, arrays |
| Speed   | Fast                  | Slower          |
| Memory  | Automatic             | Managed by GC   |
| Size    | Limited               | Large           |

## 21. foreach Loop with Arrays

### What is `foreach`?

`foreach` is a loop specially designed to **iterate through arrays and collections** easily.

In simple words:

> It reads **each element directly**, without using index.

---

### Syntax:

```csharp
foreach(dataType variable in arrayName)
{
    // use variable
}
```

---

### Example: Using foreach with Array

```csharp
int[] numbers = {10, 20, 30, 40};

foreach(int num in numbers)
{
    Console.WriteLine(num);
}
```

Output:

```
10
20
30
40
```

---

### How `foreach` Works Internally

* Starts from first element
* Automatically moves to next element
* Stops at last element
* No index handling needed

---

### Key Rules of foreach

* Read-only loop
* Cannot modify array elements
* No index access

❌ Not allowed:

```csharp
foreach(int num in numbers)
{
    num = 100; // error
}
```

---

### When to Use foreach

* When you only want to **read values**
* When index is not required
* When code readability matters

---

### foreach vs for Loop

| Feature      | foreach        | for               |
| ------------ | -------------- | ----------------- |
| Index access | ❌ No           | ✅ Yes             |
| Modification | ❌ No           | ✅ Yes             |
| Simplicity   | Very simple    | Moderate          |
| Best for     | Reading values | Logic-based tasks |

---

### Example: When NOT to use foreach

If you need index or modification:

```csharp
for(int i = 0; i < numbers.Length; i++)
{
    numbers[i] *= 2;
}
```

---

## 22. Final Interview Tip (Arrays)

> Q: Can we modify array using foreach?

Answer:
"No. `foreach` is read-only. To modify elements, we must use a `for` loop."

> Q: Where are arrays stored in C#?

Answer:
"The array object is stored in heap memory, while the reference to the array is stored in stack memory."

> Q: Difference between 2D array and jagged array?

Answer:
"In a 2D array, all rows have equal number of columns, whereas in a jagged array, each row can have different length."

> Q: Why index starts from 0?

Answer:
Because array index represents **offset from starting memory location**.

## Key Takeaways

* Array stores multiple values
* Index starts from 0
* Size is fixed
* `.Length` gives array size
* Loops + arrays = powerful combo
