## 1. What is OOP?

**OOP (Object-Oriented Programming)** is a programming approach where software is designed using **objects that represent real-world entities**.

Instead of writing long step-by-step instructions, OOP organizes code into meaningful objects like:
- Student  
- Car  
- BankAccount  
- Employee  
- Mobile  

Each object contains:
- Its own data  
- Its own behavior  

This makes programs structured and easy to manage.

---

## 2. Why OOP Was Introduced?

Before OOP, programmers used **procedural programming** (functions + variables only).

When software became large (banking systems, operating systems, websites), problems started.

### Problems in Procedural Programming
- Code became very long and messy  
- Hard to understand large programs  
- Difficult to maintain  
- Low reusability  
- One change could affect whole program  
- Hard for multiple developers to work together  

So a better way to organize programs was needed.  
That solution became **Object-Oriented Programming (OOP)**.

---

## 3. Basic Idea Behind OOP

Main idea of OOP:

> Break a big problem into small manageable objects.

Each object:
- Represents something real  
- Has its own responsibility  
- Works independently  
- Can interact with other objects  

This makes software:
- Organized  
- Easy to debug  
- Easy to expand  
- Easy to maintain  

---

## 4. Real-World Thinking in OOP

OOP follows real-life thinking.

Example: College system  
We naturally think about:
- Student  
- Teacher  
- Course  
- Exam  

Each becomes an object in program.

This makes OOP logical and natural.

## 5. Basic Terminologies in OOP

### Class
A class is a blueprint or design.

Example:
- Blueprint of house = Class  
- Actual house = Object  

Class defines:
- Data (properties)
- Behavior (methods)

But class itself is just a design, not real.

---

### Object
An object is a real instance created from a class.

Example:
- Class = Car  
- Objects = BMW, Audi, Honda  

Objects actually exist and use memory.

---

### Properties (Data)
Properties store information about object.

Example:

**Student**
- Name
- Age
- Marks

**Car**
- Color
- Model
- Speed

---

### Methods (Behavior)
Methods are actions performed by object.

Example:

**Car**
- Start
- Stop
- Drive

**Student**
- Study
- Write exam
- Attend class

---

## 6. Goals of OOP

OOP was created to achieve:

- Modularity (divide program into parts)  
- Reusability (use code again)  
- Maintainability (easy to fix/update)  
- Security (protect data)  
- Scalability (program can grow easily)
